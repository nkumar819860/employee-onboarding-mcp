System Overview

🎯 What is this System?

The Employee Onboarding MCP System is a comprehensive microservices-based platform that automates and streamlines the employee onboarding process. It consists of multiple MCP (Model Context Protocol) servers and an intelligent agent broker that orchestrates the entire workflow.

📊 System Components

🤖

Employee Onboarding Agent Broker

Intelligent orchestration engine that coordinates the entire onboarding process

💼

Assets Allocation MCP Server

Manages IT asset allocation, tracking, and returns for new employees

📧

Email Notification MCP Server

Handles all email communications throughout the onboarding process

👥

Employee Profile MCP Server

Manages employee profiles, data, and onboarding status tracking

🌟 Key Features

  • Automated Workflow: Streamlined onboarding process from start to finish
  • Asset Management: Intelligent IT asset allocation and tracking
  • Email Communications: Automated notifications and welcome messages
  • Profile Management: Comprehensive employee data management
  • Status Tracking: Real-time onboarding progress monitoring
  • Fallback Support: Robust error handling with mock mode capabilities

🤖 Employee Onboarding Agent Broker

🧠 Natural Language Processing

Use Natural Language: "Onboard a new employee named John Smith with email john.smith@company.com for the IT department, allocate laptop and ID card"

📝 Service Description

The Agent Broker is the intelligent orchestration layer that coordinates the entire employee onboarding process. It communicates with all MCP servers to complete onboarding tasks seamlessly.

🌐 Base URL

http://localhost:8081

🔗 API Endpoints

GET /api/health - Health check endpoint
GET /api/mcp/info - MCP server information
POST /api/mcp/tools/orchestrate-employee-onboarding - Complete onboarding process
POST /api/mcp/tools/get-onboarding-status - Check onboarding status
POST /api/mcp/tools/retry-failed-step - Retry failed onboarding step
GET /api/mcp/tools/check-system-health - System health check

💼 Assets Allocation MCP Server

🧠 Natural Language Processing

Use Natural Language: "Allocate a laptop and ID card to employee EMP001" or "Return asset LAP-001 in good condition"

📝 Service Description

Manages IT asset allocation, tracking, and returns for employees. Supports laptops, ID cards, mobile phones, monitors, and headsets with intelligent inventory management.

🌐 Base URL

http://localhost:8083

🔗 API Endpoints

GET /api/health - Health check endpoint
GET /api/mcp/info - MCP server information
POST /api/mcp/tools/allocate-assets - Allocate assets to employee
POST /api/mcp/tools/return-asset - Return allocated asset
GET /api/mcp/tools/list-assets - List all assets
GET /api/mcp/tools/get-available-assets - Get available assets
POST /api/mcp/tools/get-employee-assets - Get assets for employee

🏗️ System Architecture

🎯 Microservices Architecture

The Employee Onboarding MCP System follows a distributed microservices architecture with an intelligent agent broker coordinating multiple specialized MCP servers.

📊 System Flow Diagram

Agent Broker
→
Employee MCP
→
Assets MCP
→
Email MCP

🔄 Onboarding Workflow

  1. Profile Creation: Employee profile is created in the system
  2. Asset Allocation: Required IT assets are allocated to the employee
  3. Welcome Email: Welcome email is sent to the new employee
  4. Asset Notification: Notification about asset pickup is sent
  5. Completion: Onboarding completion email is sent to all stakeholders

🗄️ Database Architecture

H2 Embedded Database

Each service uses an H2 embedded database for development and testing. Key features:

  • In-memory database for fast performance
  • Automatic table creation and data initialization
  • Fallback to mock mode if database unavailable
  • Support for production PostgreSQL (configurable)

🚀 Deployment Guide

⚠️ Prerequisites

  • Java 17 or higher
  • Maven 3.8 or higher
  • Anypoint Studio (optional for development)
  • Docker (optional for containerized deployment)

💻 Local Development Setup

1. Clone the Repository

git clone https://github.com/your-org/employee-onboarding-mcp.git
cd employee-onboarding-mcp

2. Start Individual Services

# Terminal 1 - Employee MCP Server
cd employee-onboarding-mcp-server
mvn clean mule:run

# Terminal 2 - Assets MCP Server  
cd assets-allocation-mcp-server
mvn clean mule:run

# Terminal 3 - Email MCP Server
cd email-notification-mcp-server  
mvn clean mule:run

# Terminal 4 - Agent Broker
cd employee-onboarding-agent-broker
mvn clean mule:run

🐳 Docker Deployment

1. Build Docker Images

# Build all services
docker-compose build

# Or build individual services
docker build -t employee-mcp:latest ./employee-onboarding-mcp-server
docker build -t assets-mcp:latest ./assets-allocation-mcp-server
docker build -t email-mcp:latest ./email-notification-mcp-server
docker build -t agent-broker:latest ./employee-onboarding-agent-broker

2. Start Services with Docker Compose

# Start all services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop all services
docker-compose down

☁️ CloudHub Deployment

1. Configure Properties

# Update src/main/resources/config.properties
env=CloudHub
deployment.target=cloudhub
protocol.strategy=https
db.strategy=postgresql  # For production

2. Deploy to CloudHub

# Deploy each service individually
cd employee-onboarding-mcp-server
mvn clean deploy -DmuleDeploy

cd ../assets-allocation-mcp-server  
mvn clean deploy -DmuleDeploy

cd ../email-notification-mcp-server
mvn clean deploy -DmuleDeploy

cd ../employee-onboarding-agent-broker
mvn clean deploy -DmuleDeploy
