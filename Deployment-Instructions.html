🔧 Troubleshooting

Common Deployment Issues

1. Authentication Errors
Issue: Maven deploy fails with authentication error
Solution: Update Maven settings.xml with Anypoint credentials
2. Resource Limits
Issue: Insufficient vCore capacity
Solution: Use 0.1 vCores per application or upgrade subscription
3. Port Conflicts
Issue: Application fails to start due to port conflicts
Solution: Ensure unique application names in CloudHub

Log Analysis

Check application logs in Runtime Manager:

  1. Go to Runtime Manager → Applications
  2. Click on your application name
  3. Navigate to "Logs" tab
  4. Look for startup errors or health check failures

Network Connectivity

Verify service connectivity:

  • Ensure all applications are deployed to the same region
  • Check VPC configuration if using private networks
  • Verify firewall rules allow inter-service communication
